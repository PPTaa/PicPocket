# 스크린샷 및 저장 이미지 정리 앱 MVP 디자인

작성일: 2026-04-25
기준 문서: `SCREENSHOT_SORTER_APP_BRIEF.md`
상태: 사용자 승인된 MVP 설계
플랫폼: iOS 전용
최소 타겟: iOS 18

## 1. MVP 목표

이 MVP의 목표는 수익화나 Photos 앱 정리 기능이 아니라, 사용자의 스크린샷과 저장 이미지가 실제로 자동 분류될 수 있는지 검증하는 것이다.

검증하고 싶은 핵심 가설:

- 사용자는 스크린샷과 저장 이미지 속 유용한 정보를 나중에 다시 찾고 싶어 한다.
- 앱이 최근 1년의 스크린샷과 저장 이미지를 자동으로 분석해도 사용자는 개인정보 측면에서 받아들일 수 있다.
- OCR, QR/바코드 감지, 키워드/정규식 기반 분류만으로도 MVP 수준의 유용한 자동 분류가 가능하다.
- Photos 앱을 직접 수정하지 않아도 앱 내부 보관함만으로 충분한 첫 가치를 줄 수 있다.

MVP의 한 줄 설명:

> 스크린샷과 저장한 이미지 속 쿠폰, 영수증, 배송, 티켓/예약, 주소/지도, 계좌/결제 정보를 기기 안에서 자동으로 찾아 앱 내부 보관함으로 정리한다.

## 2. MVP 범위

포함하는 기능:

- 사진 접근 권한 요청
- 선택한 사진 접근과 전체 사진 접근 모두 지원
- 전체 접근 권한을 추천하되 강제하지 않음
- 최근 1년의 스크린샷 자동 스캔
- 최근 1년의 저장 이미지 후보 스캔
- OCR 텍스트 추출
- QR 코드 및 바코드 감지
- 키워드/정규식 기반 자동 분류
- 6개 카테고리와 미분류 보관함 제공
- OCR 텍스트 검색
- 상세 화면에서 카테고리 수동 변경
- 첫 스캔 완료 요약
- 앱 내부에서만 분류 상태 저장

포함하지 않는 기능:

- Photos 앱 앨범 생성
- Photos 앱 폴더 생성
- 사진 삭제
- 사진 이동
- AdMob 광고
- 인앱 결제
- Gemma 4 E2B 온디바이스 LLM 분류
- 클라우드 AI 분석
- 서버 업로드
- Face ID 잠금
- 만료일 알림
- 중복 이미지 정리

## 3. 대상 이미지

MVP의 대상은 스크린샷과 저장 이미지다.

스크린샷:

- iOS에서 캡처된 이미지
- PhotoKit의 스크린샷 식별 기능을 우선 사용한다.
- MVP의 1순위 입력 대상이다.

저장 이미지:

- Safari, 카카오톡, 쇼핑몰, 이메일, 파일 앱 등에서 저장된 것으로 추정되는 이미지
- PhotoKit에서 완벽하게 "저장 이미지"를 판별할 수 없을 수 있으므로 후보 기반으로 다룬다.
- 최근 저장된 이미지, 가져온 이미지 성격의 이미지, 텍스트/QR/바코드가 감지되는 이미지를 우선 후보로 삼는다.

제외 대상:

- 일반 카메라 사진
- 사람, 음식, 풍경 중심 사진
- 텍스트나 코드가 거의 없는 이미지
- 비디오
- Live Photo의 동영상 요소
- 숨김 항목 및 최근 삭제된 항목

## 4. 첫 스캔 정책

기본 스캔 범위는 최근 1년이다.

이유:

- 첫 실행 속도와 실사용 가치를 함께 확보할 수 있다.
- 오래된 스크린샷까지 전부 OCR 처리하는 부담을 줄일 수 있다.
- 쿠폰, 배송, 예약, 영수증 같은 카테고리는 최근 1년 안에서 가치가 충분히 발생한다.

전체 기간 스캔:

- 결과 요약 화면 또는 설정 화면에서 사용자가 직접 실행할 수 있다.
- "전체 기간 스캔은 시간이 오래 걸릴 수 있습니다"를 명확히 안내한다.
- 전체 기간 스캔 중에도 앱이 멈춘 것처럼 보이지 않도록 진행률을 표시한다.

## 5. 카테고리

MVP 카테고리는 6개와 미분류로 제한한다.

- 쿠폰
- 영수증
- 배송
- 티켓/예약
- 주소/지도
- 계좌/결제
- 미분류 / 검토 필요

카테고리 설계 원칙:

- 사용자가 실제로 다시 찾고 싶어 하는 정보 중심으로 구성한다.
- Apple Photos의 기본 분류와 겹치는 "문서", "QR 코드" 같은 종류 분류가 아니라 생활 목적 중심으로 나눈다.
- 확신도가 낮은 항목은 억지로 분류하지 않고 미분류로 보낸다.
- 계좌/결제는 민감 카테고리로 표시한다.

## 6. 사용자 흐름

첫 실행 흐름:

1. 사용자가 앱을 실행한다.
2. 앱이 "기기 안에서 스크린샷과 저장 이미지를 분석한다"는 점을 설명한다.
3. 앱이 사진 접근 권한을 요청한다.
4. 사용자가 선택 접근 또는 전체 접근을 허용한다.
5. 앱이 최근 1년의 대상 이미지를 스캔한다.
6. 앱이 OCR, QR/바코드 감지, 분류를 진행한다.
7. 앱이 결과 요약을 보여준다.
8. 사용자는 앱 내부 보관함으로 이동한다.

일반 사용 흐름:

1. 사용자가 보관함을 연다.
2. 카테고리 카드에서 분류된 항목 수와 최근 썸네일을 본다.
3. 특정 카테고리를 열어 이미지를 확인한다.
4. 잘못 분류된 이미지는 상세 화면에서 카테고리를 수정한다.
5. 필요한 이미지는 검색으로 찾는다.

검색 흐름:

1. 사용자가 검색어를 입력한다.
2. 앱은 저장된 OCR 텍스트와 카테고리명을 기준으로 결과를 보여준다.
3. 결과는 최신순으로 보여준다.
4. 결과에서 이미지를 열면 상세 화면으로 이동한다.

## 7. 화면 구성

온보딩 화면:

- 앱이 하는 일을 짧게 설명한다.
- "사진은 서버로 업로드하지 않고 기기 안에서만 분석합니다"를 명확히 표시한다.
- 전체 접근을 추천하되, 선택한 사진만 허용해도 사용할 수 있다고 안내한다.

권한 화면:

- 시스템 권한 요청 전에 자체 설명 화면을 먼저 보여준다.
- 전체 접근을 허용하면 자동 정리가 더 잘 된다는 점을 설명한다.
- 선택 접근 상태에서는 허용된 이미지 안에서만 분류된다고 설명한다.

스캔 진행 화면:

- 현재 단계 표시: 이미지 찾는 중, 텍스트 읽는 중, 코드 감지 중, 분류 중
- 처리한 이미지 수와 전체 후보 수를 표시한다.
- 사용자가 취소할 수 있게 한다.
- 취소하더라도 이미 처리된 결과는 저장할 수 있다.

결과 요약 화면:

- 최근 1년에서 찾은 후보 수
- 자동 분류된 항목 수
- 미분류 항목 수
- 카테고리별 개수
- "전체 기간 스캔" 선택지
- "보관함으로 이동" 기본 동작

보관함 화면:

- 카테고리 카드 목록
- 각 카드에 카테고리명, 항목 수, 최근 썸네일 3개 표시
- 미분류는 별도로 눈에 띄게 표시한다.
- 계좌/결제 카테고리에는 민감 정보 표시를 둔다.

카테고리 상세 화면:

- 이미지 그리드
- 최신순 정렬
- 검색 결과에서 진입한 경우 검색어 하이라이트를 고려한다.
- 각 항목은 썸네일과 간단한 감지 힌트를 보여줄 수 있다.

이미지 상세 화면:

- 원본 이미지 미리보기
- 현재 카테고리
- 카테고리 변경 버튼
- OCR 텍스트
- QR/바코드 감지 여부
- 민감 정보 여부
- Photos 앱에서 원본 열기

검색 화면:

- OCR 텍스트 검색
- 카테고리 필터
- 최신순 결과
- 결과 없음 상태

설정 화면:

- 권한 상태
- 선택한 사진 관리로 이동
- 전체 기간 다시 스캔
- 로컬 분석 데이터 초기화
- 개인정보 설명

## 8. 아키텍처

MVP는 다음 단위로 나눈다.

PhotoLibraryService:

- 사진 권한 상태 확인
- 사진 권한 요청
- 스크린샷 fetch
- 저장 이미지 후보 fetch
- 썸네일 요청
- 원본 이미지 요청

ScanCoordinator:

- 첫 스캔과 재스캔 흐름 관리
- 진행률 상태 관리
- 취소 처리
- 이미 처리한 항목 스킵

VisionAnalysisService:

- OCR 텍스트 추출
- QR 코드 감지
- 바코드 감지
- 분석 실패 결과 처리

ClassificationService:

- OCR 텍스트와 감지 정보를 입력받아 카테고리 후보를 계산
- 카테고리별 점수와 신뢰도 산출
- 낮은 신뢰도는 미분류로 처리

LocalStore:

- 이미지별 분석 결과 저장
- 카테고리 저장
- 사용자의 카테고리 수정 저장
- 검색 인덱스 역할
- 스캔 상태 저장

ViewModels:

- 온보딩 상태
- 스캔 진행 상태
- 보관함 요약
- 카테고리 상세
- 검색 결과
- 이미지 상세

UI:

- SwiftUI 기반 화면
- 처리 상태와 빈 상태를 명확하게 표시
- Photos 원본을 수정하지 않는다는 점을 반복적으로 확인 가능하게 함

## 10. 분류 로직

MVP 분류는 규칙 기반 점수 방식으로 구현한다.

입력:

- OCR 텍스트
- QR/바코드 존재 여부
- 날짜 후보
- 가격 후보
- 운송장 번호 후보
- 이미지 크기와 비율
- 스크린샷 여부
- 저장 이미지 후보 여부

출력:

- 카테고리
- 신뢰도
- 감지 이유
- 민감 정보 여부

예시 규칙:

- 쿠폰: 쿠폰, 할인, 적립, 유효기간, 만료, 바코드, QR
- 영수증: 영수증, 결제, 합계, 총액, 승인번호, 주문번호
- 배송: 배송, 택배, 운송장, 송장번호, 배송조회, 출고
- 티켓/예약: 티켓, 예매, 예약, 좌석, 탑승권, 체크인, 게이트
- 주소/지도: 주소, 지도, 길찾기, 도로명, 위치
- 계좌/결제: 계좌, 이체, 입금, 은행, 카드, 결제

신뢰도 기준:

- 높은 신뢰도: 대표 키워드와 구조적 신호가 함께 감지됨
- 중간 신뢰도: 대표 키워드는 있으나 구조적 신호가 약함
- 낮은 신뢰도: 여러 카테고리 점수가 비슷하거나 신호가 부족함

낮은 신뢰도는 미분류로 보낸다.

## 11. 개인정보와 신뢰

MVP에서는 모든 분석을 온디바이스로 처리한다.

원칙:

- 스크린샷과 저장 이미지를 서버로 업로드하지 않는다.
- 클라우드 AI를 사용하지 않는다.
- Gemma 4 E2B 같은 LLM도 MVP에서는 사용하지 않는다.
- 사진 앱 원본을 삭제하거나 이동하지 않는다.
- Photos 앨범을 만들지 않는다.
- 사용자가 권한을 줄인 경우 제한된 범위에서만 동작한다.
- 계좌/결제 카테고리는 민감 정보로 표시한다.

권한 안내 문구 방향:

> 선택한 사진만 허용해도 사용할 수 있어요. 전체 접근을 허용하면 앱이 스크린샷과 저장 이미지를 자동으로 찾아 더 정확하게 정리할 수 있어요. 사진은 서버로 업로드하지 않고 기기 안에서만 분석합니다.

## 12. 오류 처리와 예외 상태

사진 권한 없음:

- 보관함 대신 권한 안내 화면을 보여준다.
- 선택 접근도 가능하다고 안내한다.

선택 접근만 허용:

- 선택된 이미지 안에서만 스캔한다.
- 더 많은 이미지를 분석하려면 사진 선택을 수정할 수 있게 한다.

후보 이미지 없음:

- "최근 1년에서 정리할 스크린샷이나 저장 이미지를 찾지 못했습니다" 상태를 보여준다.
- 전체 기간 스캔 또는 직접 선택을 제안한다.

OCR 실패:

- 해당 이미지는 QR/바코드와 메타데이터만으로 분류를 시도한다.
- 분류가 어렵다면 미분류로 보낸다.

iCloud 원본 다운로드 필요:

- 썸네일 우선 표시
- 원본 분석이 필요한 경우 대기 또는 건너뛰기 처리
- 네트워크가 필요한 상태임을 과하게 강조하지 않고 자연스럽게 재시도한다.

스캔 취소:

- 이미 분석된 결과는 저장한다.
- 사용자가 나중에 이어서 스캔할 수 있게 한다.

저장소 오류:

- 사용자에게 분석 결과 저장에 실패했다고 알린다.
- 원본 사진에는 영향이 없음을 안내한다.

## 13. 테스트 전략

단위 테스트:

- 카테고리별 키워드 분류
- 신뢰도 계산
- 미분류 처리
- 민감 정보 판단
- 검색 인덱스 매칭

통합 테스트:

- 스캔 세션 생성부터 결과 저장까지의 흐름
- 권한 상태별 흐름
- 취소 후 재스캔 흐름
- 사용자가 카테고리를 수정했을 때 재분석 결과가 덮어쓰지 않는지 확인

수동 테스트:

- 전체 사진 접근
- 선택한 사진 접근
- 스크린샷만 있는 라이브러리
- 저장 이미지 후보가 많은 라이브러리
- iCloud 원본이 로컬에 없는 이미지
- OCR 품질이 낮은 이미지
- 한국어/영어가 섞인 이미지
- QR/바코드만 있는 이미지

성능 테스트:

- 최근 1년 후보 100개
- 최근 1년 후보 500개
- 최근 1년 후보 1,000개
- 전체 기간 스캔
- 스캔 중 앱 백그라운드 전환

## 14. 성공 기준

MVP 성공 기준:

- 사용자가 첫 스캔을 완료할 수 있다.
- 최근 1년의 스크린샷과 저장 이미지 후보를 찾을 수 있다.
- 주요 카테고리 6개 중 최소 일부가 자동으로 의미 있게 채워진다.
- 사용자가 검색으로 원하는 이미지를 찾을 수 있다.
- 잘못된 분류를 사용자가 직접 수정할 수 있다.
- 사진 앱 원본을 수정하지 않아도 사용자가 정리된 느낌을 받을 수 있다.

추적할 지표:

- 사진 권한 허용률
- 전체 접근 허용률
- 선택 접근 사용률
- 첫 스캔 시작률
- 첫 스캔 완료율
- 후보 이미지 수
- 자동 분류 성공 수
- 미분류 비율
- 카테고리 수정률
- 검색 사용률
- 3일 재방문
- 7일 재방문

## 16. 승인된 결정 요약

- MVP 목표는 자동 분류 검증이다.
- 카테고리는 6개와 미분류로 시작한다.
- 대상은 스크린샷과 저장 이미지다.
- 첫 스캔은 최근 1년 기본, 전체 기간은 옵션이다.
- Photos 앨범 생성은 MVP에서 제외한다.
- 사진 삭제와 이동은 MVP에서 제외한다.
- OCR 텍스트 검색은 MVP에 포함한다.
- 광고는 MVP에서 제외한다.
- 권한은 선택 접근과 전체 접근을 모두 지원하되 전체 접근을 추천한다.
- Gemma 4 E2B는 추후 고급 기능으로 남긴다.
